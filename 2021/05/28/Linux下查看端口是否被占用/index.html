Linux 下查看端口是否被占用

Author: Jamza Date: May 28, 2021  14:47:00 Category: 技术杂文

netstat 命令

在 Linux 下，netstat 命令用于显示网络状态，包括与 IP、TCP、UDP、ICMP 协议相关的统计数据，一般用于校验本机各个端口的网络连接情况。

命令 netstat 的输出分为了两个部分，分别是 Active Internet connections (w/o servers)，与 Active UNIX domain sockets (w/o servers)。

Active Internet connections (w/o servers) 被称为有源 TCP 连接，其中的 Recv-Q 与 Send-Q 指的是接收队列与发送队列，这些数字正常情况下应该为 0，如果非 0 则表示软件包正在队列中堆积。

Active UNIX domain sockets (w/o servers) 被称为有源 Unix 域套接口。

Proto 显示连接使用的协议，RefCnt 表示连接到本套接口上的进程数量，Type 显示套接字的类型，State 显示套接字的当前状态，Path 显示连接到套接字的其他进程使用的路径名。

状态说明：

  1. LISTEN：监听来自远端的 TCP 端口连接请求
  2. ESTABLISHED：表示一个打开的连接
  3. SYN-SENT：在发送连接请求后，等待匹配的连接请求
  4. SYN-RECEIVED：在收到和发送一个连接请求后，等待对方连接请求的确认
  5. CLOSE-WAIT：等待从本地用户发送的连接中断请求
  6. CLOSING：等待远程 TCP 对连接中断的确认
  7. CLOSED：没有任何连接状态

选项参数

  1. -a 或者 --all：显示所有连接状态中的套接口
  2. -n 或者 --numeric：直接使用 IP 地址，而不通过域名服务器
  3. -p 或者 --programs：显示正在使用套接口的程序 PID 与程序名称
  4. -r 或者 --route：显示 routing table
  5. -s 或者 --statistics：显示网络工作信息统计表
  6. -t 或者 --tcp：显示 TCP 传输协议的连接状况
  7. -u 或者 --udp：显示 UDP 传输协议的连接状况
  8. -v 或者 --verbose：显示指令执行过程

检测端口是否被占用

通过常用的命令 netstat -anp | grep 端口号，查询当前的端口被哪个进程所占用。
